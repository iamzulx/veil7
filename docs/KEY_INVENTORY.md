# Key Inventory — veil7

> **Version:** 1.0
> **Effective Date:** 2026-06-14
> **Review Cycle:** Quarterly or upon algorithm change

---

## 1. Overview

veil7 uses **exclusively ephemeral keys** — generated fresh per iteration from
OS entropy, used once, then zeroized. There are **no long-term keys**,
**no key files**, **no key databases**, and **no key rotation schedules**.

This document inventories all key types used by veil7, their lifecycle,
and protection mechanisms.

## 4. Key Protection Mechanisms

| Mechanism | Implementation | Coverage |
|-----------|---------------|----------|
| Memory locking | `mlock()` via `Locked<N>` | Master seed |
| Volatile zeroization | `zeroize_bytes()` + `compiler_fence(SeqCst)` | All secret buffers |
| ZeroizeOnDrop | Upstream `zeroize` crate | ML-KEM/ML-DSA keys |
| `#[inline(never)]` | All custom `Drop` impls | Prevents compiler elision |
| `#![deny(unsafe_code)]` | Crate-level | Limits unsafe to l0_memlock only |
